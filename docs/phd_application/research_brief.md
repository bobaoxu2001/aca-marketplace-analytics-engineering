# Concise PhD Application Research Brief

## Metric-Grounded Analytics Agents: Measuring Semantic Reliability Beyond Executable SQL

Natural-language analytics agents can produce executable SQL while silently
using the wrong measure, grain, grouping, population, or result structure. In
this independent project, I built a reproducible DuckDB/dbt warehouse over six
official CMS Plan Year 2026 Marketplace files containing **3,887,640 source
rows**, designed a 30-question benchmark, and implemented an evaluation pipeline
that separates metric routing, query compilation, policy validation, execution,
strict result agreement, and evidence reporting.

The methodological contribution is the failure decomposition and retained audit
trail. During evaluation, I discovered that the original
alias/projection-compatible score could overstate exact correctness by accepting
incomplete outputs. I therefore introduced a strict complete-result contract
requiring identical output columns, row cardinality, and cell values, then
rescored the saved runs. The pipeline records prompts, metric definitions,
permitted tables, SQL, policy checks, executed rows, evidence, and failure
classes, exposing intermediate artifacts for post-hoc failure localization.

In an **oracle-table-conditioned subscription-Codex pilot**—the model received
the question-specific allowed-table list but no gold SQL or gold rows—all 90
generated queries executed, but only **3/90 (3.3%)** strictly reproduced the
complete reference result; 17/90 matched a more permissive compatible
projection. A benchmark-specific, oracle-routed hand-authored compiler—the
compiler received the correct metric label—matched **24/30** original questions,
but only **12/30** on a hash-locked, model-generated lexical-shift diagnostic
even with oracle routing. These results show, in this benchmark, that execution
is a weak proxy for semantic correctness and that routing alone does not solve
compilation and output-contract generalization.

The evidence is diagnostic rather than confirmatory. The benchmark and compiler
were co-developed, the derived evaluation sets were generated by Codex and have
not been independently validated, the subscription interface did not expose a
stable model snapshot, a second gold-SQL audit remains pending, and the prepared
human-review packet has no completed ratings. Accordingly, the project is a
**manuscript-in-preparation research prototype**, not a general model comparison
or medical decision-support system.

My proposed doctoral agenda is to develop independently validated cross-schema
benchmarks and output contracts, learn metric/schema grounding with calibrated
abstention, and conduct human studies of whether evidence-first interfaces
improve analysts' failure detection and calibrated reliance.
